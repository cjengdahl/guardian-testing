# AI Code Assistant Security & Code-Generation Guidelines

- Add secure, testable defaults for network I/O (timeouts, certificate validation), restrict permitted URL schemes, and use secure temporary-file idioms.
- Update generated code patterns to use reusable helpers (e.g., TimeoutSession), URL-scheme validators, and tempfile APIs instead of hardcoded paths.
- Enforce these rules in CI (Bandit + unit tests) and pre-commit hooks; require documented exceptions for any deliberate weakening (e.g., verify=False).
- Provide templates and examples so the assistant always emits secure code by default and makes insecure choices explicit, reviewed, and gated.
- Add unit tests that assert presence of timeouts, certificate validation, scheme checks, and use of tempfile APIs.

## Purpose
This file defines mandatory security and coding guidelines for the AI Code Assistant tool so generated code will not reintroduce vulnerabilities flagged by previous scans (requests without timeout, verify=False, file:///custom URL schemes, and hardcoded temp directories). All code generated by the assistant must follow these rules.

## Required Secure Defaults (apply to every generated snippet)
- Network requests must always set a timeout. Use a configurable default (e.g., 10 seconds) and a reusable session wrapper so callers cannot omit timeouts accidentally.
- Certificate validation must be enabled by default. Never emit verify=False. If using private or self-signed CAs, emit an explicit verify path argument (or document secure steps to add CA to trust store).
- Only allow explicit, validated URL schemes where user input controls a URL or remote resource. Default whitelist: http, https.
- Use Python's tempfile module for temporary files and directories. Never hardcode /tmp, C:\temp, or other predictable or environment-specific temp directories.
- All deviations (exceptions) must be explicitly documented inline with code and require a security review; exceptions must be gated by an environment flag that is disabled in CI and production.

## Code-generation Rules (musts / must-not)
- MUST include a default timeout constant and use it for all requests.
- MUST provide or use a session wrapper that enforces the timeout by default.
- MUST not generate requests calls with verify=False.
- MUST validate inbound URLs and only accept allowed schemes. When performing local file operations, do not accept file:// URLs from untrusted sources.
- MUST use tempfile.NamedTemporaryFile, tempfile.TemporaryDirectory, or tempfile.mkstemp() for any temporary file needs and ensure secure file permissions where appropriate.
- MUST include unit tests demonstrating that the timeout and certificate validation are present.
- MUST include inline comments explaining secure decisions for non-obvious choices (e.g., custom CA bundles).
- MUST include a short example of secure usage in docstrings for helper utilities.

## Helper templates (copyable examples)
These templates are the canonical patterns the assistant must use or adapt when generating network or file-related code.

- Default configuration and TimeoutSession
```python
# config.py
DEFAULT_REQUEST_TIMEOUT = 10  # seconds, tune per use-case but require non-zero timeout

# requests_helpers.py
import requests
from typing import Optional, Tuple

# Allow either float or (connect, read) tuple
RequestTimeout = Optional[float]  # or Tuple[float, float]

class TimeoutSession(requests.Session):
    """Session that enforces a default timeout for all requests unless explicitly overridden."""
    def __init__(self, default_timeout: RequestTimeout = DEFAULT_REQUEST_TIMEOUT):
        super().__init__()
        self._default_timeout = default_timeout

    def request(self, method, url, **kwargs):
        kwargs.setdefault("timeout", self._default_timeout)
        # Certificate validation must remain enabled by default. Use kwargs['verify'] to pass CA bundle path when needed.
        if "verify" in kwargs and kwargs["verify"] is False:
            raise ValueError("verify=False is not allowed. Provide a CA bundle path or set verify=True.")
        return super().request(method, url, **kwargs)

# Usage
# from requests_helpers import TimeoutSession
# session = TimeoutSession()
# resp = session.get("https://example.com")  # timeout enforced
```

- Secure request example with custom CA bundle (no verify=False)
```python
# When interacting with a server using a private CA, configure verify with the CA bundle path:
session = TimeoutSession()
resp = session.get("https://internal.example", verify="/etc/ssl/certs/private-ca.pem")
```

- URL validation (whitelist schemes)
```python
from urllib.parse import urlparse

ALLOWED_URL_SCHEMES = {"http", "https"}

def validate_remote_url(url: str) -> str:
    """
    Validate that the URL uses an allowed scheme and has a host.
    Raises ValueError if validation fails.
    """
    parsed = urlparse(url)
    scheme = parsed.scheme.lower()
    if scheme not in ALLOWED_URL_SCHEMES:
        raise ValueError(f"Unsupported URL scheme: {scheme!r}; allowed: {ALLOWED_URL_SCHEMES}")
    if not parsed.hostname:
        raise ValueError("URL must include a hostname")
    return url  # return normalized/validated URL
```

- Secure temp-file usage
```python
import tempfile
import os
from contextlib import contextmanager

@contextmanager
def secure_temp_file(suffix="", prefix="tmp", mode="w+b"):
    """Creates a secure temp file; caller is responsible for cleanup if needed."""
    with tempfile.NamedTemporaryFile(suffix=suffix, prefix=prefix, delete=True, mode=mode) as tf:
        # set restrictive permissions if a persistent file is created
        try:
            os.chmod(tf.name, 0o600)
        except Exception:
            # Safe to ignore if running on a platform that doesn't support chmod the same way
            pass
        yield tf  # file-like object
```

## Unit test expectations and examples
- Generated code must include unit tests (pytest style) that assert:
  - Requests are issued with a timeout value.
  - verify is not False on requests unless a documented exception is used in code (exception must fail CI review).
  - URL validator rejects non-whitelisted schemes.
  - Temporary files are created via tempfile and not via hardcoded paths.

Example test skeleton:
```python
def test_timeout_enforced(monkeypatch):
    from requests_helpers import TimeoutSession
    s = TimeoutSession(default_timeout=5)
    called = {}
    def fake_request(method, url, **kwargs):
        called['timeout'] = kwargs.get('timeout')
        called['verify'] = kwargs.get('verify', True)
        class R: pass
        return R()
    monkeypatch.setattr(s.__class__, 'request', fake_request)
    s.get("https://example.com")
    assert called['timeout'] == 5
    assert called['verify'] is True or called['verify'] is None

def test_url_validation_rejects_file_scheme():
    from requests_helpers import validate_remote_url
    import pytest
    with pytest.raises(ValueError):
        validate_remote_url("file:///etc/passwd")
```

## CI, Static Analysis, and Pre-commit
- Add Bandit to CI and pre-commit. Configure Bandit to fail the build on:
  - use of requests without timeout,
  - requests with verify=False,
  - use of file:// or other disallowed schemes when detected,
  - hardcoded temporary directory paths (heuristic).
- Example CI step (GitHub Actions snippet):
```yaml
- name: Install security tools
  run: pip install bandit
- name: Run Bandit
  run: bandit -r . -lll -ii
```
- Example .pre-commit-config.yaml entry:
```yaml
repos:
  - repo: https://github.com/PyCQA/bandit
    rev: stable
    hooks:
      - id: bandit
        args: ['-r', '.']
```
- Add unit tests for the above rules and fail CI if tests detect missing timeouts, verify=False, disallowed URL schemes, or hardcoded temp directories.

## Exception / Opt-out Process
- Any code that would otherwise violate these rules must:
  1. Include an inline TODO comment explaining the justification.
  2. Be accompanied by a ticket/PR description that documents risk and duration (must be short-lived).
  3. Be gated by an environment-only flag (e.g., ALLOW_INSECURE_REQUESTS) that defaults to disabled; CI and production must not enable this flag.
  4. Require explicit security reviewer approval before merging.

Example guard for development-only insecure calls:
```python
import os
ALLOW_INSECURE = os.getenv("ALLOW_INSECURE_REQUESTS") == "1"
if not ALLOW_INSECURE:
    raise RuntimeError("Insecure requests prevented; set ALLOW_INSECURE_REQUESTS=1 only in local dev with approval")
# If allowed, still prefer setting a CA bundle path rather than verify=False
```

## Developer guidance for AI generation behavior
- When the assistant generates code that performs network I/O, file downloads, or accepts external resource URLs it MUST:
  - Use the TimeoutSession or equivalent wrapper.
  - Call validate_remote_url before using the URL.
  - Use tempfile.* APIs for any temporary storage.
  - Not emit verify=False; if a CA bundle is required, generate code that accepts a path and documents how to create/use the bundle.
- If the assistant is asked to produce insecure examples for education, those examples must clearly state they are insecure, provide safe alternatives, and should not be included in production-oriented code snippets.

## Audit and Review
- Every change that affects these behaviors must include automated tests and a Bandit report in CI.
- Periodically (quarterly) run a repository-wide security scan and update these guidelines and templates based on new findings.

## Short checklist for code authoring (for quick copy into assistant prompts)
- Use TimeoutSession or set timeout per request.
- Never use verify=False; provide CA bundle path if needed.
- Validate URL schemes (http/https only by default).
- Use tempfile APIs; do not hardcode temp directories.
- Add unit tests asserting these rules.
- Document and gate any exceptions with a security review.
