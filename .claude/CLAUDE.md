# AI Code Assistant Security & Code-Generation Guidelines

These instructions define mandatory security and coding guidelines for the AI Code Assistant tool. They ensure generated code does not reintroduce vulnerabilities such as requests without timeouts, disabled certificate validation (verify=False), acceptance of disallowed URL schemes (e.g., file://), or hardcoded temporary directories.

## Purpose
All code generated by the assistant MUST follow these rules to prevent recurrence of vulnerabilities flagged by static/security scans (e.g., Bandit). Generated snippets must be secure by default, testable, and include documented, gated exceptions for deliberate deviations.

## High-level rules (enforced for every generated snippet)
- Always enforce request timeouts via a default constant and a reusable session wrapper.
- Always validate remote URLs against an explicit whitelist (default: http, https). Reject file:// and other non-whitelisted schemes.
- Certificate validation must be enabled by default. Never emit verify=False. If a private/self-signed CA is required, accept and document a CA bundle path (verify="/path/to/ca.pem").
- Use Python tempfile APIs (NamedTemporaryFile, TemporaryDirectory, mkstemp) for any temporary-file needs. Never hardcode /tmp, C:\temp, or similar paths.
- Any deviation must be explicitly documented inline, accompanied by a short justification, and gated by a disabled-by-default environment flag that CI/production will not enable.

## Required Secure Defaults
- DEFAULT_REQUEST_TIMEOUT: non-zero default (recommended 10 seconds).
- TimeoutSession: a requests.Session subclass that enforces default timeout and disallows verify=False.
- ALLOWED_URL_SCHEMES: default whitelist = {"http", "https"}.
- Temporary files/dirs: use tempfile.* APIs and set restrictive permissions where appropriate.
- Exceptions/opt-outs: must use ALLOW_INSECURE_REQUESTS env flag (default off) and require security reviewer approval before merging.

## Code-generation MUST / MUST NOT (concise)
- MUST define and use a DEFAULT_REQUEST_TIMEOUT constant.
- MUST provide/use a TimeoutSession wrapper that sets timeout by default.
- MUST NOT generate requests calls with verify=False.
- MUST validate inbound URLs with an explicit whitelist; reject file:// and local file schemes when using untrusted input.
- MUST use tempfile.* for temp files/directories; no hardcoded temp paths.
- MUST include unit tests asserting presence of timeouts, certificate validation, scheme checks, and tempfile usage.
- MUST annotate non-obvious secure choices in inline comments.
- MUST document any exception with TODO, PR/ticket link, and gating env flag usage.

## Templates and Helper Examples

- Default configuration and TimeoutSession
```python
# config.py
DEFAULT_REQUEST_TIMEOUT = 10  # seconds; non-zero required

# requests_helpers.py
import os
from typing import Optional, Tuple
from urllib.parse import urlparse
import requests

RequestTimeout = Optional[float]  # or Tuple[float, float]
ALLOWED_URL_SCHEMES = {"http", "https"}

class TimeoutSession(requests.Session):
    """
    Session that enforces a default timeout and disallows verify=False.
    Use kwargs['verify'] to pass a CA bundle path when needed.
    """
    def __init__(self, default_timeout: RequestTimeout = DEFAULT_REQUEST_TIMEOUT):
        super().__init__()
        self._default_timeout = default_timeout

    def request(self, method, url, **kwargs):
        # Enforce timeout default
        kwargs.setdefault("timeout", self._default_timeout)

        # Disallow disabling certificate validation
        if "verify" in kwargs and kwargs["verify"] is False:
            # Prevent insecure behavior in generated code
            raise ValueError("verify=False is not allowed. Provide a CA bundle path or omit verify to use system trust store.")

        return super().request(method, url, **kwargs)
```

- URL validation (whitelist schemes)
```python
# url_validation.py
from urllib.parse import urlparse
ALLOWED_URL_SCHEMES = {"http", "https"}

def validate_remote_url(url: str) -> str:
    """
    Validate that the URL uses an allowed scheme and includes a hostname.
    Raises ValueError on invalid URLs. Rejects file:// and other non-whitelisted schemes.
    """
    parsed = urlparse(url)
    scheme = (parsed.scheme or "").lower()
    if scheme not in ALLOWED_URL_SCHEMES:
        raise ValueError(f"Unsupported URL scheme: {scheme!r}; allowed: {ALLOWED_URL_SCHEMES}")
    if not parsed.hostname:
        raise ValueError("URL must include a hostname")
    # Optionally normalize/return the original validated URL
    return url
```

- Secure request example with custom CA bundle (no verify=False)
```python
# usage_example.py
from requests_helpers import TimeoutSession
from url_validation import validate_remote_url

session = TimeoutSession()
url = validate_remote_url("https://internal.example")
# When interacting with servers using a private CA, pass the CA bundle path explicitly (do not use verify=False)
resp = session.get(url, verify="/etc/ssl/certs/private-ca.pem")
```

- Secure temp-file usage
```python
# tempfile_helpers.py
import tempfile
import os
from contextlib import contextmanager

@contextmanager
def secure_temp_file(suffix="", prefix="tmp", mode="w+b"):
    """
    Creates a secure temporary file using tempfile.NamedTemporaryFile.
    Caller should not hardcode temp directories. Permissions are tightened where possible.
    """
    with tempfile.NamedTemporaryFile(suffix=suffix, prefix=prefix, delete=True, mode=mode) as tf:
        # If a persistent file is created later, prefer 0o600 permissions
        try:
            os.chmod(tf.name, 0o600)
        except Exception:
            # Non-fatal: some platforms may not support chmod semantics the same way
            pass
        yield tf
```

## Unit Test Requirements (pytest examples)
All generated network- or file-related code MUST include pytest-style tests that assert the secure defaults.

- Timeout and verify tests
```python
# tests/test_requests_helpers.py
import pytest
from requests_helpers import TimeoutSession
import types

def test_timeout_enforced(monkeypatch):
    s = TimeoutSession(default_timeout=5)
    called = {}

    # Monkeypatch the Session.request on the parent class to capture kwargs passed
    def fake_request(self, method, url, **kwargs):
        called['timeout'] = kwargs.get('timeout')
        called['verify'] = kwargs.get('verify', True)
        class R: pass
        return R()

    monkeypatch.setattr(s.__class__, 'request', fake_request, raising=False)
    s.get("https://example.com")
    assert called['timeout'] == 5
    # verify should be True or omitted (None treated as system default)
    assert called['verify'] is True or called['verify'] is None

def test_disallow_verify_false():
    s = TimeoutSession()
    with pytest.raises(ValueError):
        s.get("https://example.com", verify=False)
```

- URL validation test
```python
# tests/test_url_validation.py
import pytest
from url_validation import validate_remote_url

def test_url_validation_rejects_file_scheme():
    with pytest.raises(ValueError):
        validate_remote_url("file:///etc/passwd")
```

- Tempfile usage test
```python
# tests/test_tempfile_helpers.py
import pytest
import tempfile
import tempfile_helpers
from unittest import mock

def test_secure_temp_file_uses_tempfile(monkeypatch):
    called = {}
    def fake_named_tempfile(*args, **kwargs):
        called['called'] = True
        return tempfile.NamedTemporaryFile(*args, **kwargs)
    monkeypatch.setattr(tempfile, 'NamedTemporaryFile', fake_named_tempfile)
    with tempfile_helpers.secure_temp_file() as tf:
        assert called.get('called') is True
```

Notes:
- Tests should avoid network access; monkeypatch network operations as shown.
- Tests must fail CI if generated code allows verify=False, omits timeouts, accepts file://, or hardcodes temp dirs.

## CI, Static Analysis, and Pre-commit Enforcement
- Add Bandit to CI and pre-commit. Configure Bandit and unit tests to fail the build on:
  - requests without timeout,
  - requests with verify=False,
  - use of file:// or disallowed schemes,
  - hardcoded temporary directory paths (heuristic).
- Example GitHub Actions step:
```yaml
- name: Install security tools
  run: pip install -U pip pytest bandit

- name: Run tests
  run: pytest -q

- name: Run Bandit
  run: bandit -r . -lll -ii
```
- Example .pre-commit-config.yaml snippet:
```yaml
repos:
  - repo: https://github.com/PyCQA/bandit
    rev: stable
    hooks:
      - id: bandit
        args: ['-r', '.']
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: check-added-large-files
```
- CI must run both tests and Bandit; any failure related to timeouts, verify=False, URL scheme violations, or hardcoded temp paths must block merges.

## Exception / Opt-out Process (mandatory gating)
Any deliberate deviation from these rules MUST:
1. Include an inline TODO comment in the code explaining the justification and risk.
2. Be accompanied by a short PR description or linked ticket documenting duration and mitigation.
3. Be gated by an environment-only flag ALLOW_INSECURE_REQUESTS (default disabled). CI and production MUST NOT enable this flag.
4. Require explicit security reviewer approval before merging.

Example guard:
```python
import os
ALLOW_INSECURE = os.getenv("ALLOW_INSECURE_REQUESTS") == "1"
if not ALLOW_INSECURE:
    raise RuntimeError(
        "Insecure requests prevented; to allow in local dev you must set ALLOW_INSECURE_REQUESTS=1 and obtain security approval. Prefer using a CA bundle path instead of verify=False."
    )
# If allowed, prefer setting a CA bundle path rather than verify=False
```

## Guidance for the AI Code Assistant
When generating code that does network I/O, file downloads, or accepts resource URLs the assistant MUST:
- Use TimeoutSession or an equivalent wrapper that enforces timeouts.
- Call validate_remote_url before using any input-derived URL.
- Use tempfile.* APIs for temporary storage.
- Never emit verify=False; if a CA bundle is required, accept a path argument and document how to supply it.
- Generate unit tests that assert these behaviors.
- Add inline comments explaining secure choices or reasons for alternative behaviors.

If the user explicitly requests insecure examples for educational purposes:
- Mark them clearly as insecure.
- Provide a secure alternative immediately.
- Do not include insecure examples in production-oriented code.

## Audit, Review, and Maintenance
- Changes affecting these behaviors must include unit tests and run Bandit in CI.
- Periodically run repository-wide scans and update guidelines as new vulnerabilities emerge.
- Keep templates and examples up-to-date and concise.

## Short developer checklist (for generation)
- Use DEFAULT_REQUEST_TIMEOUT and TimeoutSession.
- Validate URLs with validate_remote_url.
- Never use verify=False; require CA bundle path if needed.
- Use tempfile APIs; do not hardcode temp paths.
- Include pytest tests asserting timeout, verify, URL scheme, and tempfile usage.
- Gate any exceptions with ALLOW_INSECURE_REQUESTS and document them.
